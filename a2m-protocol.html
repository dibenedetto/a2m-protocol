AgentToMemory
A shared memory layer
for AI agents.

LangChain, Agno, n8n, CrewAI — each carries its own memory model. A2M is a thin, open protocol that lets any framework read and write to a shared memory store without rewriting a single agent.

Read the spec See adapters
The Problem

Every framework reinvents memory.

Today, each AI framework ships its own memory abstraction. Agents from different frameworks cannot share state, history, or knowledge — even when running inside the same workflow.

LangChain

ConversationBufferMemory

In-process Python object. Lives inside the chain, not exportable.

memory.load_memory_variables({})
Agno

AgentMemory / MemoryDb

Custom schema, PostgreSQL-backed. Tightly coupled to Agno's agent lifecycle.

agent.memory.add_user_memory(m)
n8n

No native memory

Relies on external HTTP calls or sticky data in workflow context.

$json["history"]
CrewAI

EntityMemory / ShortTerm

Chromadb-backed, crew-scoped. Not accessible outside the crew run.

crew.memory = True
AutoGen

in_memory context

Message-list context passed per-agent. Ephemeral, lost across runs.

agent.chat_messages[…]

The Solution

One interface.
Any framework.

A2M defines a lightweight wire protocol — REST baseline, gRPC optional — so every agent framework talks to the same memory store through a thin adapter. No framework changes required.

Memory Model

Five memory types.

A2M classifies every entry by its role and lifetime. Agents choose the right type; the store handles retention and indexing.

⚡
working
In-flight scratchpad. Current task state. Discarded at session end.
📖
episodic
Interaction history. What happened, in order. Persistent log.
🧠
semantic
Facts and knowledge. Vector-indexed for similarity search.
🔧
procedural
Learned steps and heuristics. How to accomplish goals.
🔗
external
Pointer to an external resource — file, URL, blob, database row.
Addressing

Hierarchical namespaces.

Every entry lives in a namespace. Agents scope reads and writes to exactly the right context — no collisions, no leakage.

{app} / {workflow} / {session} / {agent}
numel / wf-42 / sess-abc / agent-0
Any segment can be omitted to widen scope — e.g. numel/wf-42 reads everything in workflow 42, across all sessions and agents.
Entry Schema

One entry format.

Every memory entry, regardless of framework or backend, is represented identically.

JSON — A2M Entry 
{
  "id":        "018f2a3b-…",
  "key":       "user/goal",
  "namespace": "numel/wf-42/sess-abc/agent-0",
  "type":      "semantic",           // working | episodic | semantic | procedural | external
  "value":     "Build a real-time translation pipeline",
  "embedding": [0.12, -0.04, 0.87, …],  // optional — caller-provided or server-generated
  "meta": {
    "source_agent":     "agent-0",
    "source_framework": "langchain",
    "created_at":       "2025-09-01T14:22:11Z",
    "updated_at":       "2025-09-01T14:22:11Z",
    "ttl_s":            3600,   // null = permanent
    "tags":             ["user", "goal"],
    "confidence":       0.95
  }
}
Wire Protocol

REST baseline.
gRPC optional.

Any HTTP client can talk to A2M. High-performance runtimes can opt in to gRPC or WebSocket transports — same semantics, faster wire.

Method Endpoint Description
POST /a2m/v1/{namespace}/entries Write or upsert a memory entry
GET /a2m/v1/{namespace}/entries/{key} Read a single entry by key
GET /a2m/v1/{namespace}/entries?type=&tag=&limit= List and filter entries
POST /a2m/v1/{namespace}/query Semantic search — body: { text, type?, top_k }
DEL /a2m/v1/{namespace}/entries/{key} Delete an entry
WS /a2m/v1/{namespace}/subscribe?pattern= Real-time event stream — { event, entry } on write/delete
HTTP — Write an entry 
POST /a2m/v1/numel/wf-42/sess-abc/agent-0/entries
Content-Type: application/json

{
  "key":   "user/goal",
  "type":  "semantic",
  "value": "Build a real-time translation pipeline",
  "meta":  { "tags": ["user", "goal"], "confidence": 0.95 }
}
HTTP — Semantic query 
POST /a2m/v1/numel/wf-42/query
Content-Type: application/json

{
  "text":   "what does the user want to build?",
  "type":   "semantic",
  "top_k":  5
}

// Response
[
  { "entry": { … }, "score": 0.94 },
  { "entry": { … }, "score": 0.81 }
]
Framework Adapters

Thin wrappers.
No framework changes.

Each adapter implements the framework's native memory interface and delegates all reads/writes to the A2M HTTP API. Adopting A2M requires zero changes to existing agents.

LangChain
⬤  In progress
Implements BaseMemory. Drop-in replacement for ConversationBufferMemory.
A2MMemory(client, namespace)
Agno
⬤  In progress
Implements MemoryDb. Wraps Agno's storage interface with A2M calls.
A2MAgnoStore(client, namespace)
n8n
⬤  Ready today
HTTP Request node. No adapter code needed — A2M REST is immediately usable.
POST /a2m/v1/… from any workflow
CrewAI
⬤  Planned
Wraps BaseMemory to route crew memory through A2M.
A2MCrewMemory(client, namespace)
AutoGen
⬤  Planned
Hooks into conversation context to persist and retrieve entries via A2M.
A2MAutoGenHook(client, namespace)

Python — LangChain adapter 
from langchain.memory import BaseMemory
from a2m import A2MClient

class A2MMemory(BaseMemory):
    client: A2MClient
    namespace: str

    def save_context(self, inputs, outputs):
        self.client.write(
            self.namespace,
            key="chat_history",
            type="episodic",
            value={"in": inputs, "out": outputs}
        )

    def load_memory_variables(self, inputs):
        results = self.client.query(
            self.namespace,
            text=str(inputs),
            top_k=5
        )
        return {"history": [r.value for r in results]}
